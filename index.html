Sobre o Projeto

O FastInBox e um projeto academico de Ciencia da Computacao orientado a transformar um processo hoje fragmentado em uma plataforma digital unica para venda e operacao de marmitas personalizadas. A proposta conecta nutricionistas, pacientes, cozinha e administracao em uma trilha rastreavel, onde o pedido nasce com a nutricionista, e revisado e pago pelo paciente e segue para producao apenas quando esta apto operacionalmente.

Alem do produto em si, o projeto estrutura de ponta a ponta os artefatos de engenharia e gestao exigidos para validacao academica e continuidade tecnica: requisitos, arquitetura, modelo de negocio, plano de projeto, modelo de dados, evidencias de sprint e documentacao publica em GitHub Pages. O recorte atual prioriza um MVP demonstravel, com foco em clareza de jornada, governanca de status, experiencia white label e base evolutiva para persistencia real, analytics e escala nas proximas fases.

Diretriz de Identidade Visual

O layout foi atualizado para um modelo neo-brutal com gramatica visual direta: blocos retangulares, borda forte, sombra dura, tipografia sem ornamento e distribuicao hierarquica orientada por criticidade.

Principio de contraste

Base monocromatica para reduzir ruido cognitivo e reforcar legibilidade em leitura tecnica extensa.

Composicao estrutural

Cards e secoes operam como blocos semanticos de leitura; cada bloco representa um artefato de engenharia.

Referencial de qualidade

A referencia de benchmark foi de clareza objetiva de informacao, mantendo autoria e linguagem FastInBox.

Baseline Tecnico da Plataforma

Arquitetura orientada a camadas: front-end de experiencia, back-end de regras e repositorio de docs como camada de governanca, com evolucao incremental de acordo com cada sprint.

Aplicacao

  • Next.js 15 + React 19
  • TypeScript como base de contratos
  • Estado local persistente para MVP academico

API e dominio

  • NestJS 11 para evolucao de regras server-side
  • Modelo de entidades pronto para migracao gradual
  • Separacao por perfis e trilha de status operacional

Entrega e governanca

  • Vercel para aplicacao demonstravel
  • GitHub Pages para documentacao tecnica publica
  • Versionamento por repositorio e commits semanticos
