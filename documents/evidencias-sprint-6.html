3.1 Autoatendimento por codigo

BACK | SELF-SERVICE
Endpoint: GET /public/self-service/:code Endpoint: POST /public/self-service/:code/register Auth: PUBLICO

Resolucao publica do codigo retorna a clinica em modo whitelabel (nome, cores, nutricionista) para montar o onboarding. O cadastro cria o paciente ja vinculado a clinica e ao nutricionista do codigo. Codigo invalido responde 404 SELF_SERVICE_CODE_INVALID.

Como foi testado

  1. Spec Jest self-service.service.spec.ts cobrindo resolucao, codigo invalido e cadastro vinculado.
  2. Validacao manual via curl com codigo do seed.
  3. Conferencia do paciente criado em GET /admin/diagnostics.

Como voce testa

  1. Pegue o codigo em GET /self-service/codes (nutricionista).
  2. Resolva via GET /public/self-service/:code sem token.
  3. Cadastre um paciente via POST .../register.

3.2 Assinaturas recorrentes

BACK | SUBSCRIPTIONS
Endpoint: GET /subscriptions Endpoint: POST /subscriptions/run Ciclo: pause / resume / cancel

Listagem de assinaturas (nutricionista auto-escopado, admin ve todas) traz o estimatedTotalCents por cadencia. O POST /subscriptions/run processa as assinaturas vencidas gerando pedidos em AWAITING_PATIENT_REVIEW, cada uma isolada em try/catch.

Como foi testado

  1. Spec Jest subscriptions.service.spec.ts cobrindo cadencias, estimativa e runDue().
  2. Seed inicia 3 assinaturas ativas e roda runDue() uma vez no boot.
  3. Validacao via curl de listagem e disparo manual.

Como voce testa

  1. Login como admin e liste em GET /subscriptions.
  2. Dispare POST /subscriptions/run.
  3. Use pause/resume/cancel em /subscriptions/:id/....
$ curl -sH "Authorization: Bearer $TOKEN" http://localhost:4001/subscriptions | jq '.[] | {patientId, cadence, status, estimatedTotalCents}'

3.3 Previsao de demanda

BACK | FORECAST
Endpoint: GET /admin/reports/forecast?days=7 Auth: admin Status: 200 OK

Projeta o volume de marmitas por janela (dia + slot + regiao) no horizonte solicitado. Cada janela traz expectedMeals, a banda lowerBound..upperBound e a decomposicao fromHistory + fromSubscriptions. Invariantes garantidas: expectedMeals === fromHistory + fromSubscriptions e lowerBound <= expectedMeals <= upperBound.

Como foi testado

  1. Spec Jest forecast.service.spec.ts validando baseline, banda e invariantes.
  2. Seed com 5 pedidos historicos (3 DELIVERED + 2 PAID) + 3 assinaturas ativas.
  3. Soma das 6 janelas conferida contra totals.

Como voce testa

  1. Login como admin via POST /auth/login.
  2. Chame /admin/reports/forecast?days=7.
  3. Confira que cada janela respeita as invariantes.
$ curl -sH "Authorization: Bearer $TOKEN" "http://localhost:4001/admin/reports/forecast?days=7" | jq
{
  "generatedAt": "2026-06-23T09:14:02.551Z",
  "horizonDays": 7,
  "windows": [
    {
      "date": "2026-06-23", "weekday": "tuesday", "slot": "12:00-12:30",
      "expectedMeals": 4, "lowerBound": 3, "upperBound": 5,
      "fromHistory": 4, "fromSubscriptions": 0
    },
    {
      "date": "2026-06-25", "weekday": "thursday", "slot": "19:00-19:30", "regionCode": "0451",
      "expectedMeals": 6, "lowerBound": 5, "upperBound": 7,
      "fromHistory": 3, "fromSubscriptions": 3
    }
  ],
  "totals": { "expectedMeals": 30, "lowerBound": 24, "upperBound": 36 }
}

A resposta completa traz 6 janelas no horizonte de 7 dias; acima estao duas amostras representativas (uma puramente historica, outra com contribuicao de assinaturas). A banda 24..36 em torno de 30 reflete +/- 1 desvio padrao do baseline historico, dando margem operacional para a cozinha.

3.4 Telas em producao

FRONT | UI
Rotas: /cadastro/:code, /nutricionista/assinaturas, /admin/previsao Build: Next.js 15 + recharts Verificado em navegador

As tres telas foram renderizadas e verificadas em navegador contra o backend ao vivo.

  • /cadastro/:code - onboarding publico whitelabel que aplica as cores da clinica resolvidas pelo codigo, ajusta o contraste do texto via readableText e expoe um formulario de 8 campos para cadastro do paciente.
  • /nutricionista/assinaturas - tabela com as 3 assinaturas ativas (cadencia, status, estimativa), acoes Pausar/Retomar/Cancelar, formulario "Nova assinatura" e card "Codigo de autoatendimento" com link publico copiavel.
  • /admin/previsao - KPIs (30 esperadas / banda 24-36 / 6 janelas), grafico recharts com banda de confianca (Area) sobre a linha de esperado e tabela de janelas, alem do botao "Rodar cobrancas vencidas".
