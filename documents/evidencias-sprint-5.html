5. Hardening aplicado

ResponsaveisBackThiago Lucas AlvesSegurancaGabriel Pahl
  • CORS restrito: origin: process.env.FRONTEND_URL + credentials. Negativa explicita para qualquer outra origem.
  • JWT short-lived: access token 15min, refresh token 7d com rotacao a cada uso.
  • HMAC SHA-256 timing-safe: webhooks comparados com crypto.timingSafeEqual para evitar timing attacks.
  • Janela anti-replay: webhook descarta payload com timestamp +/- 5 minutos fora do agora.
  • RawBody capture: Nest com { rawBody: true } para preservar bytes originais na assinatura.
  • Domain errors estruturadas: UnauthorizedDomainError, ForbiddenDomainError, ConflictDomainError, ValidationDomainError com codigos auditaveis.
  • Validacao em linha: DTOs com validators e GlobalHttpExceptionFilter centralizando resposta de erro.
